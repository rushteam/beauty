# statesync —— 基于 pkg/gameloop 搭状态同步 + AOI 骨架

和 [`examples/gameloop`](../gameloop)(帧同步)共用同一个 `pkg/gameloop.Room`,
**只换了 `OnTick` 的策略**,就从"下发输入"变成"下发状态":

|  | 帧同步 (lockstep) | 状态同步 (state sync) |
|---|---|---|
| 谁算 | 客户端各自算(要确定性) | 服务器权威算 |
| 下发什么 | **输入**(全体玩家本帧输入) | **状态**(实体坐标,AOI 过滤后) |
| 示例 | `examples/gameloop` | 本目录 |

## 这个示例做了什么

- **权威模拟**:`World.Step` 每帧把玩家移动输入应用到世界坐标(服务器说了算);
- **AOI(兴趣区域)**:每帧建一个 `pkg/spatial` 网格索引;每个连接在**出口**用
  `index.Nearby(自己的坐标, 视野半径)` 只取可视实体,下发给该客户端;
- **扇出模型**:`Room` 每帧只广播一份"权威世界"(`*worldTick`,内部对象),
  各连接的写循环把它**投影**成各自的 `View`(AOI 过滤 + 可序列化)再发。
  即"内部全量扇出 + 出口按连接过滤",状态同步 + AOI 的常见做法。

## 运行

```bash
go run ./examples/statesync
```

出生点故意摆成:alice(50,50) 与 bob(120,120) 相距 ≈99 < 视野 100 → 互相可见;
carol(800,800) 远处 → 谁也看不见它。3 个 bot 连上后各收自己的 AOI 视图并自校验:

```
──────── AOI(状态同步)校验 ────────
  alice  视野内: [bob]            (期望 [bob]) ✅
  bob    视野内: [alice]          (期望 [alice]) ✅
  carol  视野内: []               (期望 []) ✅
结论: ✅ 每个客户端只收到视野内实体(AOI 过滤生效)
```

## 复用到的 beauty 原语

| 原语 | 作用 |
|---|---|
| `pkg/gameloop` | 定步长 tick + 输入聚合 + 权威世界扇出 + 生命周期 |
| `pkg/spatial` | 每帧网格索引,`Nearby` 做 AOI 视野过滤 |
| `pkg/ws` | 连接:读循环 `Push` 输入、写循环投影 AOI 视图后写回 |

> 生产化提示:本示例每连接在出口做 AOI 过滤(内部全量扇出),连接数很大时应改为
> 服务器侧按玩家算增量、每连接独立队列下发,并配紧凑二进制(protobuf 等)与按房间
> 粘连路由。高频动作类还需 UDP/KCP 替代 WebSocket(TCP)。这些示例未涉及。
